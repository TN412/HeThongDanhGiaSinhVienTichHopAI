🎯 CUSTOM EVENTS (9 TYPES)

   1. ai_chat_request
      → Student asks AI during assignment
      Properties: submissionId, questionId, promptLength, studentId
      Measurements: promptLength

   2. ai_chat_error
      → AI chat fails
      Properties: submissionId, errorType, errorMessage, studentId

   3. submission_submitted
      → Student submits assignment
      Properties: submissionId, assignmentId, studentId, questionsCount
      Measurements: totalScore, aiSkillScore, finalScore, timeSpentMinutes

   4. assignment_generated
      → Instructor generates assignment
      Properties: assignmentId, instructorId, questionType, documentType
      Measurements: questionsGenerated, generationTime

   5. openai_api_call
      → OpenAI API call (also tracked as dependency)
      Properties: endpoint, success, model, submissionId
      Measurements: duration, tokens (prompt, completion, total)

   6. http_request
      → All HTTP traffic
      Properties: route, method, statusCode, userId, userRole
      Measurements: duration, requestSize

   7. http_error
      → HTTP failures (4xx, 5xx)
      Properties: route, method, statusCode, errorType
      Measurements: duration

   8. slow_request
      → Requests taking >5 seconds
      Properties: route, method, statusCode, userId
      Measurements: duration

   9. slow_database_query
      → MongoDB queries taking >1 second
      Properties: query
      Measurements: duration

🚀 QUICK START

   1. Create Application Insights Resource
      - Azure Portal > Create Resource > Application Insights
      - Or: az monitor app-insights component create ...

   2. Get Connection String
      - Overview page > Connection String
      - Or: az monitor app-insights component show ...

   3. Configure Backend
      - Add to .env:
        APPINSIGHTS_CONNECTION_STRING=InstrumentationKey=...;...

   4. Start Backend
      - npm run dev
      - Check console for:
        ✅ Application Insights initialized successfully
        📊 Environment: development
        📊 Memory tracking started (every 5 minutes)

   5. Verify in Azure Portal
      - Navigate to App Insights > Live Metrics
      - Send test request: curl http://localhost:5000/api/health
      - See request appear in Live Metrics Stream

   Expected Usage:
   - Small app (~100 req/min): ~1-2 GB/month → FREE ✅

   - Large app (~10k req/min): ~50-100 GB/month → $100-200/month

   Cost Optimization:
   - Enable sampling (keep 10-20% of data)
   - Disable console log collection
   - Adjust retention (90 days → 30 days)
   - Filter out noisy routes (/health)
